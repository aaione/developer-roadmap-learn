- 判断原因：读者的问题更像“怎么在浏览器或 Node.js 里跑一个本地模型”，需要按接入步骤、验证标准和工程边界展开。

- 写作重点：保留 developer-roadmap 对浏览器、Node.js、无需云服务、文本生成、情感分析、翻译、Wasm、低延迟、隐私和离线能力的介绍；补充 pipeline 验证、模型加载、缓存、WebGPU 和端侧推理边界。
- 参考内容：
  - Hugging Face Transformers.js 文档：Transformers.js 面向浏览器和 Node.js，复用 Transformers 生态的任务和模型。
  - Pipelines 文档：pipeline 把任务、模型、Tokenizer、预处理和后处理封装成高层接口，适合最小验证。
  - Vanilla JavaScript 教程：前端页面可以直接加载库并运行模型，但首次加载和交互状态要设计。
  - WebGPU 指南：WebGPU 可改善部分端侧推理性能，但要有设备支持和回退策略。
  - Environment Variables 文档：模型来源、缓存和远程加载策略会影响隐私、性能和可复现性。
  - npm 包页面：`@huggingface/transformers` 是 JavaScript 侧安装入口，适合 Node.js 或前端构建链路接入。

### Open AI Embedding Models

- 文章类型：Explanation B。
- 判断原因：读者需要理解 OpenAI Embedding Models 为什么能把文本变成可检索的语义坐标，再知道如何选模型和落地。
- 输出文件：`docs/ai-engineer/12-local-runtime-embeddings/open-ai-embedding-models.md`
- 写作重点：保留 developer-roadmap 对 Embeddings API、数值向量、语义搜索、聚类和相似度比较的介绍；补充切分、元数据过滤、评估集、模型成本和向量数据库连接方式。
- 参考内容：
  - OpenAI Embeddings Guide：Embedding 可用于搜索、聚类、推荐、异常检测和分类，核心是把文本映射成可比较的向量。
  - Embeddings API Reference：API 接收输入和模型参数，返回 embedding 向量，适合封装成文档入库和查询链路。
  - OpenAI 新 embedding 模型公告：`text-embedding-3-small` 和 `text-embedding-3-large` 在成本、维度和质量之间提供不同取舍。
  - OpenAI Cookbook 语义搜索示例：语义搜索通常先生成文档向量，再对查询生成向量并按相似度排序。
  - OpenAI Cookbook 向量数据库示例：生产链路要把向量存入数据库，并结合元数据、索引和召回策略。
  - OpenAI context window 帮助文档：Embedding 检索常用于给模型提供上下文，但上下文窗口仍然有容量边界。

- 文章类型：How-to Guide。
- 判断原因：读者的问题是“怎样在本地选模型、运行模型并接进应用”，需要给出可执行顺序和验证标准。
- 输出文件：`docs/ai-engineer/12-local-runtime-embeddings/ollama-models.md`
- 写作重点：保留 developer-roadmap 对本地运行 LLM、个人设备、隐私、性能、离线、Ollama SDK 和低延迟的介绍；补充模型库选型、本地 API、Modelfile、资源约束和生产边界。
- 参考内容：
  - Ollama 官网：Ollama 面向本地运行模型，强调在个人设备上拉取和运行模型。
  - Ollama README：安装后可以拉取模型并通过命令行运行，适合作为本地模型起步入口。
  - Ollama API 文档：本地 HTTP API 支持生成、聊天、模型管理和 embeddings 等能力。
  - Ollama Modelfile 文档：Modelfile 能固定模型来源、系统提示词、参数、模板和适配器。
  - Ollama LangChain Python 教程：本地模型可以作为应用链路的一部分接入上层框架。
  - Ollama Library：模型库提供不同模型家族、规模和标签，是选模型的入口。
