## 参考内容与提炼

### Vector Databases

- [Pinecone Docs：Concepts](https://docs.pinecone.io/guides/get-started/concepts)
- [Cloudflare Vectorize：Vector databases](https://developers.cloudflare.com/vectorize/reference/what-is-a-vector-database/)
- [Qdrant Docs：Search](https://qdrant.tech/documentation/concepts/search/)
- [Milvus Docs：What is Milvus](https://milvus.io/docs/overview.md)
- [pgvector：Open-source vector similarity search for Postgres](https://github.com/pgvector/pgvector)

提炼：向量数据库的核心不是“存数组”，而是围绕 Embedding 提供索引、相似度检索、元数据过滤、命名空间或租户隔离、混合检索和更新能力。正文保留了原始章节关于高维向量、ANN、语义搜索、推荐和内容发现的介绍，并补充了维度匹配、权限标签、专有名词检索和索引重建这些工程问题。

### RAG

- [Google Cloud：What is Retrieval-Augmented Generation?](https://cloud.google.com/use-cases/retrieval-augmented-generation)
- [AWS：What is RAG?](https://aws.amazon.com/what-is/retrieval-augmented-generation/)
- [IBM Think：What is RAG?](https://www.ibm.com/think/topics/retrieval-augmented-generation)
- [NVIDIA Blog：What Is Retrieval-Augmented Generation?](https://blogs.nvidia.com/blog/what-is-retrieval-augmented-generation/)
- [Lewis et al.：Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks](https://arxiv.org/abs/2005.11401)

提炼：RAG 把检索和生成拆开，让模型在回答前拿到可更新、可追溯的外部资料。正文保留原始章节关于“先检索相关数据，再基于数据生成答案”的核心介绍，并补充了查询改写、混合搜索、重排、引用、校验、过期资料清理和评估集。

### AI Agents

- [Anthropic Engineering：Building effective agents](https://www.anthropic.com/engineering/building-effective-agents)
- [OpenAI Agents SDK：Agents](https://openai.github.io/openai-agents-python/agents/)
- [LangChain Docs：Agents](https://docs.langchain.com/oss/javascript/langchain/agents)
- [Google ADK：Agents](https://adk.dev/agents/)
- [Microsoft Learn：AI Agent Orchestration Patterns](https://learn.microsoft.com/en-us/azure/architecture/ai-ml/guide/ai-agent-design-patterns)

提炼：Agent 的重点是 LLM 能在工具、计划和环境反馈之间循环，而不是只生成一次答案。Anthropic 对 workflow 和 agent 的区分很有帮助：路径清晰时优先工作流，步骤难预测时再上 Agent。正文保留原始章节关于“感知环境、决策、采取行动”的定义，并补充了工具边界、停止条件、权限隔离和人工确认。
